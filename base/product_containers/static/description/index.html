The module allows for efficient management and tracking of containers and pallets as products.
The containers are physical products used to store and ship other products eg. Plastic, wooden or metal boxes, pallets, bags etc.
They are usually returned to the seller after use, but some containers are meant for single use only and may not be returned - like cardboard boxes.
The module is particulary useful for managing and tracking containers in which products are delivered - to be retrieved or to be sold to the customer, together with the main product.

General features:

Create container and pallet products, to put other products in containers and then containers on pallets.
Define how much product a container can carry and how many containers can go on a pallet.

Containers on orders:

Automatically add containers and pallets on sale/purchase order.
Depending on the rules you set up, the number of containers/pallets will change to cover your needs.

Keep track of containers and pallets:

Return containers and pallets to suppliers or retrieve them from customers.
Check how many containers and pallets you have on hand.

Expand the functionalities:

Use our additional modules that work in conjunction with Product Container module and extend its functionalities.
You can further costumize your rules and make packaging even easier.

Product Containers Customer:

With this module you can define specific settings per customer or vendor.
Make product packagings with different rules and choose which customer or vendor it applies on.
Those product packagings will then only be available when you're selling to or buying from those defined on it.
If you don't select any customer or vendor, the product packaging will be available for everyone.

Product Containers Tracking:

Receive incoming shipments of products and assign them serial numbers (e.g. 001) while processing it to the container.
The system will automatically create an Incoming move for the container with serial number (i.e. 001).
You can track containers at hand by locations.
Track containers with serial number once issued (together with the Lot/Serial number module).
Get exact location of containers with usage of serial numbers.

How to use the module?

How does it work?
A real-life example for use of containers are plastic boxes used for fruits and vegetables, which you can easily manage and create with this module.
First you need to create a product and define it as container, then set the quantity of products for each container.
You also have the option to configure a container to carry the product in a larger quantity or to carry different products by selecting the container product in packaging.

Pallets:
  • Define a product as pallet (e.g. EURO Pallet).
  • When you receive products you can now manually add the pallet to the Purchase order as a product.
  • If the pallet is defined as purchasable in the product settings, it will also be transfered to the supplier invoice with its purchase price.
  • That same pallet (EURO Pallet) can also be used on the Sale order as a product.
  • If it's defined as sellable in the product settings, the sale price will be used and invoiced.
  • If it's not sellable however, the price set on Sale order will be 0, and it will only be available on the warehouse OUT.
Containers:
  • First you have to define a product as container (e.g. Plastic Box) and optionally set a Pallet for it.
  • Then create a new Product packaging used for one kind of product (e.g. Plastic Box for Apples) and set Contained quantity.
  • For example: 10kg of Apples (so if you sell 25kg of apples it will automatically add 3 containers to the Sale order).
  • You can also define a container on the product variant itself.

This module is only compatible with Community version. It is not available on Enterprise (on-premise or Odoo.sh).
The module cannot be installed on Odoo Online (Cloud), since it doesn't allow third-party modules.
